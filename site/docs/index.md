# PB Insight

PB와 고객의 자산 목적에 맞는 전략을 설계하고 실행하기 위한 내부용 문서 플랫폼입니다.

이 사이트의 목적은 PB의 영업, 상담, 운용, 제안, 기록 기준을 한곳에 모아 바로 실행 가능한 구조로 정리하는 것입니다.

## 이 사이트의 목적

| 영역 | 목적 |
| --- | --- |
| 영업 | 매일 실행할 행동과 우선순위를 명확하게 정리 |
| 상담 | 고객과의 대화 구조와 전달 문구를 일관되게 유지 |
| 운용 | 투자 철학, 랩 제안, 판단 기준을 문서화 |
| 기록 | 반복 가능한 PB 운영 체계를 축적 |

## 한눈에 보기

- `Only for PB`: 하루 실행 보드
- `BEST PB MIND`: 상위 PB 마인드셋 기준
- `PB's WRAP`: 랩 제안 철학과 상품 구조 요약
- `PB Tier System`: 고객 응대 우선순위 기준
- `PB's Pick / Idea`: 투자 아이디어와 판단 프레임
- `Weekly Report`: 주간 점검 루틴

## 이 사이트를 어떻게 쓰는가

1. 아침에는 `Only for PB`로 하루 우선순위를 정한다.
2. 상담 전에는 `BEST PB MIND`와 `PB's WRAP`으로 메시지를 정리한다.
3. 고객 관리 기준은 `PB Tier System`으로 맞춘다.
4. 투자 아이디어는 `PB's Pick`과 `PB's Idea`에 누적한다.
5. 주말 또는 주초에는 `Weekly Report`로 리듬을 점검한다.

## 안내 페이지

### ONLY FOR PB

- 문서: [Only for PB](ai-presentation/only-for-pb.md)
- 설명: 하루 행동 목표, 우선 고객, 시간 루틴, 핵심 원칙을 한 화면에 모아 둔 집중 보드입니다.

### BEST PB MIND

- 문서: [BEST PB MIND](ai-presentation/best-pb-mind.md)
- 설명: 상위 PB가 가져야 할 태도와 고객 인식 구조를 정리한 기준 문서입니다.

### PB'S WRAP

- 문서: [PB's WRAP](ai-presentation/pbs-wrap.md)
- 설명: 랩 제안 철학, 상품 구조, 고객 설명 포인트를 정리한 문서입니다.

### PB TIER SYSTEM

- 문서: [PB Tier System](ai-presentation/pb-tier-system.md)
- 설명: 고객 티어 기준, 서비스 차등, 응대 우선순위를 정리합니다.

### PB'S PICK

- 문서: [PB's Pick](ai-presentation/pbs-pick.md)
- 설명: 섹터, 종목, 상품 아이디어와 투자 판단 기준을 정리합니다.

### PB'S IDEA

- 문서: [PB's Idea](ai-presentation/pbs-idea.md)
- 설명: 리서치 기반 운용 아이디어와 상담 연결 포인트를 공유합니다.

### PB INSIGHT STYLE

- 문서: [PB Insight Style](ai-presentation/pb-insight-style.md)
- 설명: 문서 구조와 메시지 톤을 어떤 방식으로 가져갈지 정리합니다.

### WEEKLY REPORT

- 문서: [Weekly Report](ai-presentation/pb-weekly-report.md)
- 설명: 매주 반복 점검할 실적, 고객, 파이프라인 체크 항목을 정리합니다.

## 운영 원칙

- 많이 아는 것보다 많이 실행하는 것이 우선이다.
- 모든 고객보다 자금이 움직일 고객에 집중한다.
- 상담 없는 영업보다 상담으로 이어지는 영업을 만든다.
- 기록이 기억보다 강한 경쟁력이다.
- 단기 실적과 장기 신뢰 사이의 균형을 지킨다.
