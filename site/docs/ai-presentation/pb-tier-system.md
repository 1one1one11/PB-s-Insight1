# PB Tier System

PB 고객 응대와 운영 우선순위를 일관되게 유지하기 위한 내부 기준 초안입니다.

## 1) 티어 목적

- 고객 자산 규모보다 `필요 서비스의 복잡도`와 `관계 유지 중요도`를 함께 본다.
- 티어는 차별이 아니라 `응대 수준 표준화`를 위한 장치다.
- 모든 티어에 대해 기본적인 신뢰, 응답, 기록 원칙은 동일하다.

## 2) 운영 티어 정의

| Tier | 기준 | 핵심 서비스 | 기본 SLA |
| --- | --- | --- | --- |
| S | 초고액/핵심 관계 고객 | 포트폴리오 전략, 수시 콜백, 맞춤 브리핑 | 당일 |
| A | 고액/성장 가능 고객 | 정기 점검, 리밸런싱 제안, 이슈 브리핑 | 24시간 내 |
| B | 일반 관리 고객 | 정기 안내, 상품 설명, 체크리스트 제공 | 48시간 내 |
| C | 정보성/초기 접점 고객 | 기본 안내, 니즈 탐색, 분류 작업 | 72시간 내 |

## 3) 서비스 차등 원칙

### 상담 빈도

- S: 필요 시 즉시
- A: 월 1회 이상 정기 점검
- B: 분기 단위 점검
- C: 이벤트 발생 시 기본 접촉

### 정보 제공 형태

- S/A: 고객 맞춤형 브리핑
- B: 템플릿 기반 요약 자료
- C: 입문형 설명 자료

## 4) 티어 재분류 조건

- 자산 유입 또는 이탈
- 상담 반응률 및 관계 강도
- 투자 니즈 복잡도 변화
- 장기 관리 가치 상승

## 5) 운영 체크리스트

1. 고객별 최신 티어가 CRM과 메모에 일치하는지 확인
2. 최근 30일간 미응대 고객이 있는지 점검
3. 상위 티어 고객의 리스크 이벤트 대응 여부 확인
4. 티어 기준 변경 시 문서와 현장 운영을 동시에 수정
