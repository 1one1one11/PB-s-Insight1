# PB Insight Style

앞으로 문서를 어떤 스타일로 개선하고 유지할지에 대한 기준입니다.

## 1) 문서 구성 원칙

- 설명보다 `바로 쓰는 구조`를 우선한다.
- 한 문서에는 하나의 결론과 하나의 실행 흐름을 둔다.
- 문서는 읽는 자료가 아니라 `상담과 판단을 돕는 도구`여야 한다.

## 2) 기본 포맷

| 구역 | 목적 |
| --- | --- |
| 요약 | 핵심 메시지 3줄 |
| 체크리스트 | 판단 누락 방지 |
| 표 | 비교와 우선순위 정리 |
| 다음 액션 | 실제 실행 연결 |

## 3) 문장 스타일

- 추상적 표현보다 기준과 조건을 쓴다.
- "좋다"보다 "어떤 상황에서 유효한가"를 쓴다.
- 예측 문장보다 준비 문장을 우선한다.

## 4) 문서 개선 루프

1. 실제 상담에서 썼는지 확인
2. 읽고 바로 행동할 수 있는지 점검
3. 중복 표현 제거
4. 표와 체크리스트로 재정리
